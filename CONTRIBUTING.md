# Contributing to Sweep

Thank you for your interest in contributing! This project welcomes pull requests and issues from the community. Please take a moment to read through the following guidelines before submitting changes.

## Getting Started

1. Fork the repository and create a new branch from `main`.
2. Install dependencies with `pnpm install`.
3. Run `pnpm dev` to start the development server.
4. Make your changes, keeping code style consistent with the existing codebase.

## Development Checklist

- [ ] Lint passes (`pnpm lint`).
- [ ] Production build succeeds (`pnpm build`).
- [ ] Any new functionality includes test coverage or manual verification notes.
- [ ] Documentation updates accompany user-facing changes.

## Pull Requests

- Keep pull requests focused; submit separate PRs for unrelated changes.
- Provide a clear description of the change and the motivation behind it.
- Reference related issues in the PR description (e.g., `Closes #123`).
- Be prepared to iterate based on reviewer feedback.

## Commit Guidelines

- Follow conventional commit messages where possible (e.g., `feat:`, `fix:`, `docs:`).
- Use present tense and concise language.
- Squash commits when requested by maintainers.

## Reporting Issues

- Search existing issues to avoid duplicates.
- Clearly describe the problem and provide reproduction steps.
- Include information about your environment (OS, Node.js version, browser).

## Code Style

- This project relies on ESLint and TypeScript for consistency.
- Prefer functional components and hooks.
- Add comments sparingly and only where clarity is needed.

Thank you for helping make Sweep better! If you have questions, open an issue or reach out to `support@johuniq.tech`.
